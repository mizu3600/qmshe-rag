# 非图 RAG 基线与 QMSxE 三种方法对比

## 1. 实验口径

- 数据：与扩大测试集实验完全相同的 HotpotQA 固定划分，测试集 288 条。
- 随机种子：涉及已训练 Query LoRA 或 Reranker 的方法使用 13、42、73 三个种子；确定性基线只运行一次。
- 检索对象：每条样本上下文中的同一批事实文本，不使用图邻接关系。
- 候选预算：60；最终报告 Recall@10/20/40、Hit@10、Complete@20 和 MRR。
- `Hit@10` 表示前 10 项至少命中一条金证据，可视作检索层面的二元 accuracy；它不是答案生成 Accuracy/EM。
- 三种 QMSxE 方法采用此前 Stage A+Stage B（A1B1）的相同结果。
- 本实验只比较检索质量，没有调用答案生成 LLM，因此不报告答案 EM/F1。

方法定义：

- **BM25**：纯词法检索，无图、无神经 reranker。
- **Vanilla Dense**：原始 BGE-M3 向量余弦检索，无图、无 reranker。
- **BM25 + Dense Hybrid**：BM25 与 Dense 排名的 RRF 融合，无图、无 reranker。
- **Tuned Query Dense**：使用 Stage B 训练后的 Query LoRA 查询表示，无图、无 reranker。
- **Dense + Tuned Reranker**：Dense 召回 60 个候选，再用 Stage C 训练后的 reranker 排序。这是本组最强的非图基线。

## 2. 非图基线结果

| 方法 | R@10 | R@20 | R@40 | Hit@10 | Complete@20 | MRR |
|---|---:|---:|---:|---:|---:|---:|
| BM25 | 0.3620 | 0.5506 | 0.9143 | 0.6389 | 0.2604 | 0.3114 |
| Vanilla Dense (BGE-M3) | 0.7523 | 0.8855 | 0.9789 | 0.9792 | 0.7500 | 0.7139 |
| BM25 + Dense Hybrid | 0.5744 | 0.8226 | 0.9780 | 0.8993 | 0.6215 | 0.5293 |
| Tuned Query Dense | 0.7541±0.0013 | 0.8891±0.0009 | 0.9807±0.0000 | 0.9815±0.0020 | 0.7558±0.0020 | 0.7349±0.0010 |
| BM25 + Tuned Reranker | 0.8744±0.0035 | 0.9603±0.0009 | 0.9953±0.0000 | 1.0000±0.0000 | 0.9028±0.0000 | 0.8358±0.0024 |
| **Dense + Tuned Reranker** | **0.8744±0.0035** | **0.9613±0.0012** | **0.9970±0.0000** | **1.0000±0.0000** | **0.9062±0.0000** | **0.8358±0.0024** |

简单 RRF Hybrid 没有超过纯 Dense：当前 BM25 排名相对较弱，等权融合反而稀释了 Dense 的排序。加入强 reranker 后，BM25、Dense 和 Hybrid 的结果几乎收敛，说明最终得分主要由 reranker 决定。

## 4. 结论

1. **普通图确实大幅强于弱基线。** Entity-Relation 相比 Vanilla Dense，R@20 高 0.0720，MRR 高 0.1224；这证明完整 QMSxE 管线有实际效果，但不能把全部增益归因于图结构。
2. **当前最强结果并不是图方法独占。** Dense + Tuned Reranker 的 R@20 为 0.9613，Entity-Relation 为 0.9575；两者差异不显著。Entity-Relation 的 MRR 高 0.00044，同样不显著。可把二者表述为统计持平，不能宣称普通图击败强非图 RAG。
3. **Reranker 是本轮最大增益来源。** Vanilla Dense 加同一 tuned reranker 后，R@20 从 0.8855 升到 0.9613，MRR 从 0.7139 升到 0.8358。Query LoRA 单独只把 R@20 从 0.8855 提到 0.8891。
4. **超图在当前实现和数据上仍明显落后。** 它不仅落后两种普通图，也在 R@20、Complete@20、MRR 上显著落后强非图基线。这与此前发现的 gate 坍缩、粗粒度 mention 超边、角色不足和 HotpotQA 偏二元路径等问题一致。
5. **现有测试有候选饱和效应。** 每条样本平均事实数量约 42，而候选预算为 60；reranker 通常能看到几乎全部事实。因此不同召回器产生的候选差异被抹平，最终结果主要衡量 reranker，而不是图检索结构。

## 5. 可以安全写入论文/汇报的表述

> 在 288 条固定 HotpotQA 测试样本上，QMSxE Entity-Relation 显著优于无 reranker 的 BM25 和 BGE-M3 Dense 基线；但在使用相同训练后 reranker 的公平条件下，其 R@20、Complete@20 和 MRR 与强 Dense RAG 基线无统计显著差异。Reified-Fact 的 R@20 和 Complete@20 略低于该强基线，当前超图实现则显著落后。结果表明普通图管线具有竞争力，但现阶段的主要性能增益来自强 reranker，尚不能证明谱图结构本身优于强非图检索。

## 6. 下一轮应补的关键实验

- 将候选预算固定为 5、10、20、40，观察图结构在严格候选约束下是否带来优势。
- 建立跨样本的全局语料库检索，而不是每题只在 oracle context 内检索。
- 对所有方法做 reranker off/on 的完整二因素消融，避免把 reranker 增益误归因于图。
- 单独统计 bridge/comparison、1-hop/2-hop，以及路径完整率和桥梁事实召回率。
- 同时报告延迟、候选数、索引规模和增量更新时间，判断接近的准确率是否值得额外图开销。
